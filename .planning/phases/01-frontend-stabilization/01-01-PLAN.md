# Phase 1 Plan: Frontend Stabilization

## Objective

Make the rewrite workflow reliable by tracking exact selection ranges, synchronizing the returned document after updates, and fixing error presentation.

## Tasks

- Track selection state as `{ start, end, text }`
- Replace substring-based apply logic with range-based apply logic
- Sync local document state from `updateDocument(...)`
- Unify dismissible error handling for load, AI, and update failures

## Verification

- Rewrite applies to the intended occurrence
- Version metadata updates after apply
- Conflict prevention still blocks stale writes
